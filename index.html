Introdução

  • O processo de teste tem dois objetivos distintos:
    • Demonstrar ao desenvolvedor e ao cliente que o software atende a seus requisitos. Isso significa é necessário realizar ao menos um teste por requisito.
    • Descobrir situações em que o software se comporta de maneira incorreta, indesejável ou de forma diferente das especificações.
  • Os testes fazem parte, portanto, da fase de validação e verificação do software:
    • Validação: ‘estamos construindo o produto certo?’
    • Verificação: ‘estamos construindo o produto da maneira certa?’

Testes de desenvolvimento

  • Incluem todas as atividades de testes que são realizadas pela equipe de desenvolvimento do sistema.
  • Existem 2 tipos básicos de testes de desenvolvimento:
    1. Teste unitário - é o processo de testar a funcionalidade cada componente do programa, como métodos ou classes de objetos.
    2. Teste de sistema - também conhecido como teste de integração, verifica se os componentes do sistema são compatíveis e interagem corretamente.

Testes de release

  • O objetivo dessa fase de teste é demonstrar que o sistema cumpre todos os requisitos.
  • O sistema é testado como um todo a partir de diferentes perspectivas, usando:
    • Testes de requisitos
    • Testes de cenários
    • Testes de desempenho

Testes de usuário

  • É um estágio no processo de teste em que os clientes fornecem conselhos e opiniões sobre o sistema.
  • Existem 3 tipos básicos de teste de usuário:
    1. Teste alfa - os usuários trabalham com a equipe de desenvolvimento auxiliando na descoberta de defeitos antecipadamente.
    2. Teste beta - uma versão do software é disponibilizada aos usuários para que eles usem e encontrem bugs.
    3. Teste de aceitação - os clientes testam o sistema com o objetivo de decidir se ele está ou não pronto para ser implantado.
